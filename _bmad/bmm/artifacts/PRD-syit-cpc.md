# 产品需求文档 (PRD): 沈阳工业大学计算机程序设计社团官方网站

## 1. 文档概述

### 1.1 项目背景
沈阳工业大学计算机程序设计社团目前完全依赖微信群进行信息发布和成员交流，导致资讯分散、难以沉淀，成员学习成果无法有效展示。本项目旨在打造一个社团专属的官方网站系统，建立完整的成员成长体系和内容审核机制。

### 1.2 目标
- 建立社团自主的数字化社区平台
- 实现成员成长体系数字化（预备成员→正式成员→负责人）
- 建立内容发布与审核机制
- 沉淀社团技术知识库

### 1.3 范围
**MVP（第一阶段）**: 成员管理、转正考核、资讯/博客发布与审核、基础内容展示

### 1.4 定义与术语
| 术语 | 定义 |
|------|------|
| 预备成员 | 刚加入社团，尚未通过考核的成员 |
| 正式成员 | 通过考核，拥有内容审核权限的成员 |
| 负责人 | 社团管理层，拥有最高权限 |
| 转正考核 | 预备成员通过答题+项目评审成为正式成员的过程 |
| 20%通过制 | 内容审核机制，超过正式成员总数20%同意即通过 |

## 2. 用户角色与权限

### 2.1 角色定义

#### 2.1.1 预备成员（约50人）
- **描述**: 刚加入社团的学生，正在学习阶段
- **核心需求**: 了解社团、学习技术、申请转正

**权限矩阵:**
| 功能 | 权限 |
|------|------|
| 浏览公开内容 | ✅ |
| 发布内容 | ✅（需审核） |
| 参与审核 | ❌ |
| 成员管理 | ❌ |
| 申请转正 | ✅ |

#### 2.1.2 正式成员（约100人）
- **描述**: 通过考核的社团核心成员
- **核心需求**: 发布内容、参与审核、展示作品

**权限矩阵:**
| 功能 | 权限 |
|------|------|
| 浏览公开内容 | ✅ |
| 发布内容 | ✅（需审核） |
| 参与内容审核投票 | ✅ |
| 成员管理 | ❌ |
| 审批转正申请 | ❌ |

#### 2.1.3 负责人
- **描述**: 社团管理层（社长、副社长等）
- **核心需求**: 管理成员、把控内容质量、社团运营

**权限矩阵:**
| 功能 | 权限 |
|------|------|
| 所有正式成员权限 | ✅ |
| 成员角色管理 | ✅ |
| 审批转正申请 | ✅ |
| 内容终审（直接通过/拒绝） | ✅ |
| 系统配置 | ✅ |

## 3. 功能需求

### 3.1 成员管理系统

#### 3.1.1 用户注册
**需求描述**: 新成员通过学号/邮箱注册账号

**验收标准:**
- [ ] 支持学号+姓名+邮箱注册
- [ ] 邮箱验证机制
- [ ] 初始角色为"预备成员"
- [ ] 密码强度要求：至少8位，包含字母和数字

#### 3.1.2 用户登录
**需求描述**: 支持多种登录方式

**验收标准:**
- [ ] 支持邮箱+密码登录
- [ ] 支持"记住我"功能（7天）
- [ ] 登录失败限制（5次失败后锁定15分钟）
- [ ] 密码找回功能

#### 3.1.3 个人信息管理
**需求描述**: 用户管理个人资料

**验收标准:**
- [ ] 可修改头像、昵称、个人简介
- [ ] 可修改密码
- [ ] 可绑定/修改邮箱
- [ ] 可查看自己的内容发布历史
- [ ] 可查看审核记录（投票历史）

#### 3.1.4 成员列表（管理员）
**需求描述**: 负责人查看和管理所有成员

**验收标准:**
- [ ] 成员列表展示（头像、姓名、角色、加入时间）
- [ ] 按角色筛选
- [ ] 搜索成员
- [ ] 修改成员角色
- [ ] 禁用/启用账号

---

### 3.2 转正考核系统

#### 3.2.1 转正申请
**需求描述**: 预备成员提交转正申请

**验收标准:**
- [ ] 申请页面展示考核说明和流程
- [ ] 提交项目作品（GitHub链接或文字描述）
- [ ] 申请状态追踪（待审核/审核中/已通过/已拒绝）

#### 3.2.2 在线答题
**需求描述**: 转正考核的答题环节

**验收标准:**
- [ ] 题库管理（负责人可维护）
- [ ] 随机抽题生成试卷
- [ ] 答题倒计时（默认60分钟）
- [ ] 自动判卷（客观题）
- [ ] 分数计算和展示

**题目类型:**
- 单选题
- 多选题
- 判断题
- 简答题（需人工评分）

#### 3.2.3 项目评审
**需求描述**: 负责人评审申请者的项目作品

**验收标准:**
- [ ] 项目提交表单（项目名称、描述、技术栈、链接）
- [ ] 负责人评分（百分制）
- [ ] 评审意见反馈
- [ ] 项目分数与答题分数合并计算

#### 3.2.4 转正审批
**需求描述**: 负责人审批转正申请

**验收标准:**
- [ ] 待审批列表
- [ ] 查看申请者答题分数和项目分数
- [ ] 设置通过分数线（默认：答题60%+项目40%，总分≥70分）
- [ ] 审批通过/拒绝操作
- [ ] 审批结果通知（邮件/站内信）

---

### 3.3 内容发布系统

#### 3.3.1 资讯发布
**需求描述**: 发布社团新闻、活动通知等

**验收标准:**
- [ ] 富文本编辑器（支持标题、段落、列表、链接、图片）
- [ ] 文章标题、摘要、封面图
- [ ] 发布时选择文章类型（资讯/博客）
- [ ] 草稿保存功能
- [ ] 预览功能

#### 3.3.2 博客发布
**需求描述**: 发布技术博客、学习心得

**验收标准:**
- [ ] 与资讯发布共用编辑器
- [ ] 支持代码块（语法高亮）
- [ ] 支持标签（可选）
- [ ] 作者信息自动关联

#### 3.3.3 内容编辑与删除
**需求描述**: 管理已发布内容

**验收标准:**
- [ ] 作者可编辑自己的内容（审核中/已通过均可编辑，重新进入审核）
- [ ] 作者可删除自己的内容
- [ ] 负责人可删除任何内容

---

### 3.4 内容审核系统

#### 3.4.1 审核队列
**需求描述**: 展示待审核的内容列表

**验收标准:**
- [ ] 待审核内容列表（标题、作者、类型、提交时间）
- [ ] 按类型筛选
- [ ] 按时间排序
- [ ] 分页展示

#### 3.4.2 审核投票
**需求描述**: 正式成员参与内容审核投票

**验收标准:**
- [ ] 内容详情页展示完整内容
- [ ] "通过" / "拒绝" 投票按钮
- [ ] 已投票状态标记
- [ ] 当前投票统计（通过数/总正式成员数）
- [ ] 超过20%通过即自动发布
- [ ] 超过20%拒绝即自动驳回

**审核规则:**
- 通过门槛：通过票数 > 正式成员总数 × 20%
- 拒绝门槛：拒绝票数 > 正式成员总数 × 20%
- 每人每篇文章只能投一票
- 负责人投票权重：直接决定通过/拒绝

#### 3.4.3 审核记录
**需求描述**: 记录审核历史

**验收标准:**
- [ ] 个人审核历史（我投过的票）
- [ ] 内容审核历史（某篇文章的审核记录）
- [ ] 审核统计（个人审核通过率等）

---

### 3.5 内容展示

#### 3.5.1 首页
**需求描述**: 网站入口，聚合展示核心内容

**验收标准:**
- [ ] 社团介绍区域
- [ ] 最新资讯列表（前5条）
- [ ] 热门博客列表（前5条）
- [ ] 快捷入口（发布内容、审核中心、个人中心）
- [ ] 社团统计数据（成员数、内容数等）

#### 3.5.2 资讯列表页
**需求描述**: 展示所有已发布的资讯

**验收标准:**
- [ ] 列表展示（封面、标题、摘要、作者、发布时间）
- [ ] 分页（每页10条）
- [ ] 按时间排序（默认最新）

#### 3.5.3 博客列表页
**需求描述**: 展示所有已发布的博客

**验收标准:**
- [ ] 列表展示（标题、摘要、作者、发布时间、阅读数）
- [ ] 分页（每页10条）
- [ ] 按时间排序（默认最新）

#### 3.5.4 内容详情页
**需求描述**: 展示单篇文章的完整内容

**验收标准:**
- [ ] 标题、作者、发布时间
- [ ] 富文本内容渲染
- [ ] 代码块语法高亮
- [ ] 阅读数统计
- [ ] 分享功能（复制链接）

#### 3.5.5 个人主页
**需求描述**: 展示成员个人信息和发布内容

**验收标准:**
- [ ] 个人资料展示（头像、昵称、简介、角色）
- [ ] 该成员发布的内容列表
- [ ] 加入时间

## 4. 非功能需求

### 4.1 性能需求
- 页面加载时间 < 2秒
- 支持并发用户 50+
- 图片懒加载

### 4.2 安全需求
- 密码加密存储（bcrypt）
- SQL注入防护
- XSS防护
- CSRF防护
- 敏感操作需确认

### 4.3 可用性需求
- 响应式设计（支持移动端访问）
- 浏览器兼容：Chrome、Firefox、Safari、Edge 最新两个版本
- 错误提示友好

### 4.4 可维护性需求
- 代码注释完整
- API文档完整
- 数据库变更脚本

## 5. 技术架构

### 5.1 技术栈
- **后端**: Spring Boot 3.x + Java 17
- **前端**: Vue.js 3 + TypeScript
- **数据库**: MySQL 8.0
- **缓存**: Redis（可选，用于会话存储）
- **部署**: Docker + Docker Compose

### 5.2 系统架构
```
┌─────────────────┐
│   Vue.js 前端   │
│   (Nginx)       │
└────────┬────────┘
         │ HTTP/REST
┌────────▼────────┐
│  Spring Boot    │
│   后端 API      │
└────────┬────────┘
         │
┌────────▼────────┐
│     MySQL       │
│    数据库       │
└─────────────────┘
```

## 6. 数据库设计（概要）

### 6.1 核心实体
- **用户** (users): id, 学号, 姓名, 邮箱, 密码, 角色, 头像, 简介, 状态, 创建时间
- **转正申请** (applications): id, 用户id, 答题分数, 项目分数, 项目描述, 状态, 申请时间
- **题目** (questions): id, 类型, 题目内容, 选项, 正确答案, 分值
- **内容** (posts): id, 标题, 摘要, 内容, 类型, 作者id, 封面图, 状态, 创建时间
- **审核记录** (reviews): id, 内容id, 审核人id, 投票类型, 创建时间
