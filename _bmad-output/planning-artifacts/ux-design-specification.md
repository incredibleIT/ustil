## Executive Summary

### Project Vision

沈阳工业大学计算机程序设计社团官方网站是一个**PC 端优先**的内容发布与交流平台。采用**内容优先布局（Content-first Layout）**设计模式，最大化内容展示区域，提供沉浸式阅读体验。该平台旨在替代微信群，建立社团专属的信息发布、成员管理和内容交流系统。

**设计原则：**
- PC 端浏览器优先
- 内容展示区域最大化
- 简洁现代的视觉风格
- 清晰的信息层次和操作引导

### Target Users

**1. 预备成员（ROLE_PROBATION）**
- 刚加入社团的新成员
- 需要通过在线答题和项目提交完成转正考核
- 主要活动：学习、提交转正申请、浏览社团内容

**2. 正式成员（ROLE_MEMBER）**
- 已通过考核的社团成员
- 可以发布资讯和博客内容
- 参与内容审核投票
- 主要活动：发布内容、审核他人内容、参与社区互动

**3. 负责人（ROLE_ADMIN）**
- 社团管理者，拥有最高权限
- 管理成员（修改角色、禁用/启用账号）
- 审批转正申请
- 管理所有内容（编辑、删除）
- 主要活动：成员管理、内容审批、转正审核

### Key Design Challenges

1. **从微信优先转向 PC 优先**
   - 当前前端过度适配微信端，导致 PC 端布局效果差
   - 需要重构所有页面的布局结构
   - 保持已有功能完整，只重构样式和布局

2. **内容优先布局实现**
   - 确保内容区域最大化，减少视觉干扰
   - 顶部极简导航，居中内容区
   - 白色背景卡片 + 圆角 + 轻微阴影

3. **设计系统重建**
   - 基于参考图片重建完整的 UI 组件库
   - 建立统一的颜色、间距、字体规范
   - 确保所有页面遵循一致的设计语言

4. **UX 文档丢失恢复**
   - 之前的 UX 设计文档意外删除
   - 需要重新创建完整的 UX 设计规范
   - 确保文档版本管理和备份

### Design Opportunities

1. **沉浸式阅读体验**
   - 利用 PC 端大屏幕优势，优化长文阅读体验
   - 合理的行高、字号、段落间距
   - 代码块语法高亮，提升技术博客阅读体验

2. **清晰的信息层次**
   - 通过卡片、间距、颜色建立清晰的视觉层次
   - 角色颜色标识（预备=灰色、正式=蓝色、负责人=金色）
   - 内容状态可视化（草稿、待审核、已发布）

3. **高效的操作流程**
   - 内容发布 3 步内完成（快速入口、极简编辑器、一键发布）
   - 审核投票 1 秒完成（Optimistic UI、实时更新）
   - 转正流程可视化（步骤条 + 百分比 + 预计时间）

4. **成长仪式感设计**
   - 转正成功小庆祝动画
   - 角色徽章授予
   - 提升用户归属感和成就感

## Core User Experience

### Defining Experience

**核心体验：浏览和阅读内容**

如果只做好一件事，那就是让用户能够**流畅、舒适地浏览和阅读社团内容**。

所有其他功能（发布、审核、转正）都是围绕这个核心体验展开的支撑功能。首页是用户的第一触点，必须做到：
- 进入首页即可看到最新内容，无需多余操作
- 内容卡片清晰展示标题、摘要、作者、时间、类型标签
- 点击标题或卡片即可进入详情页沉浸式阅读
- 导航简洁明了，快速切换到不同内容类型（资讯/博客）

### Platform Strategy

**主平台**：PC 端浏览器（桌面和笔记本）
**次要平台**：无（不强制兼容微信端）
**输入方式**：键鼠操作为主
**屏幕尺寸**：首先优化 1920x1080 和 1366x768 分辨率
**不需要考虑**：原生移动 App、离线功能、微信内置浏览器

平台策略的核心是**专注**：集中所有设计资源打造最佳的 PC 端体验，不为多平台兼容性牺牲核心体验。

### Effortless Interactions

以下交互应该做到**零思考、零学习成本**：

1. **内容浏览**
   - 进入首页即可看到最新内容
   - 滚动即可加载更多，无需手动分页
   - 点击即进入详情，无需二次确认

2. **内容发布**
   - 点击"发布内容"按钮即进入编辑器
   - 极简工具栏，常用功能一目了然
   - 自动保存草稿，防止内容丢失
   - 3 步内完成发布：点击发布 → 编辑内容 → 发布

3. **审核投票**
   - 看到内容 → 点击通过/拒绝 → 完成
   - Optimistic UI 设计，点击即刻反馈
   - 1 秒完成投票，无需跳转页面

4. **角色自适应**
   - 界面根据用户角色自动适配
   - 不同角色看到不同的功能和菜单
   - 权限边界清晰但不突兀

### Critical Success Moments

以下关键时刻决定了用户是否会认为"这个平台比微信群好用"：

1. **首次浏览首页** ⭐⭐⭐⭐⭐
   - **目标**：看到清晰的内容卡片和导航，感受到专业感
   - **设计要点**：
     - 欢迎语个性化（"欢迎回来，张三 👋"）
     - 内容卡片设计精美，信息层次清晰
     - 顶部导航简洁专业
     - 整体视觉风格现代、清爽

2. **首次发布内容** ⭐⭐⭐⭐
   - **目标**：体验到流畅的编辑和发布流程
   - **设计要点**：
     - 编辑器界面简洁，工具栏不拥挤
     - 实时预览功能
     - 发布成功有明确的反馈提示

3. **首次审核投票** ⭐⭐⭐⭐
   - **目标**：感受到简单高效的审核机制
   - **设计要点**：
     - 审核队列清晰展示待审核内容
     - 投票按钮醒目，操作简单
     - 投票结果实时更新

4. **转正通过** ⭐⭐⭐⭐⭐
   - **目标**：看到庆祝动画和角色徽章升级
   - **设计要点**：
     - 转正成功页面有庆祝动画（confetti 或类似效果）
     - 角色徽章从灰色（预备）升级为蓝色（正式）
     - 明确的成就感反馈

### Experience Principles

以下原则将指导所有 UX 设计决策：

**1. PC 优先，内容为王**
- 所有设计决策首先考虑 PC 端用户体验
- 内容展示区域最大化，减少视觉干扰
- 不强制兼容微信端，专注打造最佳 PC 体验

**2. 简洁高效，零学习成本**
- 核心操作 3 步内完成
- 界面直观，无需教程即可使用
- 减少不必要的点击和跳转

**3. 角色自适应**
- 不同角色看到不同的功能和信息
- 权限边界清晰但不突兀
- 界面根据用户状态智能调整

**4. 成长仪式感**
- 转正流程可视化
- 角色升级有庆祝反馈
- 提升用户归属感和成就感

**5. 专业感的视觉呈现**
- 简洁现代的 UI 风格
- 清晰的信息层次
- 一致的设计语言
- 精心设计的细节（圆角、阴影、间距）

## Desired Emotional Response

### Primary Emotional Goals

**主要情绪目标：专业、高效、成就、沉浸**

用户在使用沈阳工业大学计算机程序设计社团官方网站时，应该感受到以下核心情绪：

1. **专业感和信任感**
   - 首次访问时感受到这是一个专业、可靠的社团平台
   - 通过精美的视觉设计、稳定的性能、一致的设计语言建立信任

2. **高效和掌控感**
   - 发布内容、审核投票等操作时的流畅体验
   - 3 步内完成核心操作，让用户感受到"这很简单"
   - Optimistic UI 设计，操作即刻反馈，增强掌控感

3. **成就感和归属感**
   - 转正通过时的庆祝和角色升级带来的自豪感
   - 社区互动功能增强成员间的连接感
   - 角色徽章系统提供明确的成长路径

4. **沉浸和专注**
   - 阅读内容时的舒适体验
   - 内容优先布局，减少视觉干扰
   - 合理的字体、间距、代码高亮，提升阅读体验

### Emotional Journey Mapping

**用户在不同阶段的情绪变化：**

| 阶段 | 用户情绪 | 目标情绪 | 设计要点 |
|------|----------|----------|----------|
| **首次发现**（首次访问） | 好奇、期待 | 专业、信任、清晰 | 精美的首页、清晰的导航、有价值的内容展示 |
| **核心体验**（浏览/阅读） | 专注、沉浸 | 舒适、高效、有价值 | 良好的阅读体验、内容卡片设计、快速加载 |
| **完成任务**（发布/审核） | 高效、掌控 | 流畅、简单、有成就感 | 简洁的编辑器、一键投票、即时反馈 |
| **成长时刻**（转正通过） | 兴奋、自豪 | 庆祝、归属、激励 | 庆祝动画、角色徽章、明确的成功提示 |
| **持续使用**（回头客） | 熟悉、高效 | 习惯、依赖、社区感 | 个性化欢迎、内容更新通知、社区互动 |

### Micro-Emotions

**关键的微情绪设计：**

- **信心 vs 困惑** → 清晰的导航和明确的提示，建立信心
- **信任 vs 怀疑** → 专业的视觉设计、稳定的性能，建立信任
- **兴奋 vs 焦虑** → 转正成功的庆祝动画，创造兴奋感
- **成就感 vs 挫折感** → 流畅的操作流程、即时反馈，带来成就感
- **愉悦 vs 满意** → 精美的细节设计（圆角、阴影、动画），创造愉悦感
- **归属感 vs 孤立感** → 社区互动功能、角色徽章，增强归属感

### Design Implications

**情绪 → UX 设计方法连接：**

| 情绪目标 | UX 设计方法 |
|----------|-------------|
| 专业感和信任感 | 简洁现代的 UI、一致的设计语言、稳定的性能、精心设计的细节 |
| 高效和掌控感 | 3 步内完成核心操作、Optimistic UI、清晰的导航、即时反馈 |
| 成就感和归属感 | 转正庆祝动画、角色徽章、社区互动功能、成长可视化 |
| 沉浸和专注 | 内容优先布局、合理的间距和字体、代码高亮、减少视觉干扰 |
| 愉悦和惊喜 | 精美的卡片设计、平滑的过渡动画、个性化的欢迎语 |

### Emotional Design Principles

**指导情绪设计的原则：**

**1. 第一印象决定信任**
- 首页是建立专业感的关键触点
- 精心设计的欢迎语、内容卡片、导航
- 避免混乱、拥挤的布局

**2. 流畅即愉悦**
- 减少不必要的点击和跳转
- 操作即刻反馈，不让用户等待
- 自动保存、草稿恢复等贴心功能

**3. 成长需要仪式感**
- 转正流程可视化，让用户看到进步
- 角色升级有庆祝动画和明确反馈
- 徽章系统提供长期激励

**4. 社区感来自连接**
- 显示作者信息、阅读数、审核记录
- 鼓励内容互动（虽然 MVP 阶段可能只有审核投票）
- 个人主页展示成员贡献

**5. 细节决定体验**
- 圆角、阴影、间距的精心调整
- 颜色的一致性使用（角色标识、状态标识）
- 动画的微妙运用（不过度，但提升体验）

**需要避免的负面情绪：**
- ❌ 困惑（导航不清、操作不明确）
- ❌ 挫折（操作复杂、加载缓慢、错误提示不友好）
- ❌ 焦虑（不确定状态、缺乏反馈）
- ❌ 孤立（缺乏社区感、不知道其他人的存在）

## UX Pattern Analysis & Inspiration

### Inspiring Products Analysis

**1. n8n.io（自动化工作流平台）**

**UX 亮点：**
- **动态滚动效果**：页面滚动时的动态反馈，元素渐入、布局变化流畅自然
- **现代简约设计**：大量留白、清晰的信息层次、专业的视觉呈现
- **交互式演示**：通过动画展示产品功能，让用户直观理解价值
- **渐变和微动画**：按钮悬停、卡片hover 时的微妙反馈，提升精致感

**可借鉴到本项目：**
- 首页滚动时的内容卡片渐入动画
- 导航栏滚动时的微妙变化（如阴影加深）
- 按钮和卡片的 hover 效果（颜色渐变、轻微放大）
- 页面切换时的平滑过渡动画

**2. GitHub（开发者社区平台）**

**UX 亮点：**
- **清晰的信息架构**：仓库、Issues、Pull Requests 的组织方式直观高效
- **状态可视化**：通过颜色、图标清晰展示状态（open/closed/merged）
- **高效的导航**：左侧文件树、顶部标签页、面包屑导航的完美结合
- **代码阅读体验**：语法高亮、行号、代码折叠的专业实现

**可借鉴到本项目：**
- 角色标识系统（预备/正式/负责人的颜色和图标）
- 内容状态管理（草稿/待审核/已发布的视觉区分）
- 内容详情页的代码块高亮和格式优化
- 个人主页的贡献展示方式

**3. Apple.com.cn（苹果官网）**

**UX 亮点：**
- **沉浸式产品展示**：大尺寸图片、视频、滚动触发的动画效果
- **极致的细节**：每个像素都经过精心打磨，圆角、阴影、间距完美统一
- **流畅的交互**：页面切换、模态框、导航菜单的动画无比顺滑
- **内容优先**：产品图片和文案是主角，UI 元素退居次要位置

**可借鉴到本项目：**
- 首页的大尺寸欢迎区域和视觉冲击力
- 精心设计的圆角、阴影、间距系统
- 页面切换和模态框的平滑动画
- 内容展示区域的视觉优先级管理

### Transferable UX Patterns

**导航模式：**
- **顶部极简导航**（Apple、GitHub）→ 适合您的内容优先布局
- **粘性导航栏**（n8n、Apple）→ 滚动时导航栏保持在顶部，方便随时切换
- **清晰的层级导航**（GitHub）→ 面包屑 + 标签页的组合

**交互模式：**
- **滚动触发动画**（n8n、Apple）→ 内容卡片渐入、元素动态加载
- **Hover 反馈**（所有三个产品）→ 按钮、卡片、链接的微妙反馈
- **Optimistic UI**（GitHub）→ 审核投票即时反馈，无需等待服务器响应
- **平滑页面切换**（Apple）→ 路由切换时的过渡动画

**视觉模式：**
- **大量留白**（Apple、n8n）→ 营造专业感、提升可读性
- **圆角卡片 + 精致阴影**（Apple、n8n）→ 现代 Web 设计趋势
- **渐变和微动画**（n8n、Apple）→ 按钮、背景、加载状态的微妙渐变
- **大尺寸视觉元素**（Apple）→ 首屏的英雄区域、大标题、清晰的视觉焦点
- **状态颜色系统**（GitHub）→ 通过颜色快速传达状态信息

**动效模式（重点借鉴）：**
- **渐入动画**：内容卡片滚动到视口时淡入并轻微上移
- **Hover 效果**：卡片悬停时轻微放大、阴影加深、边框颜色变化
- **按钮反馈**：点击时的按下效果、加载状态的旋转动画
- **页面过渡**：路由切换时的淡入淡出或滑动效果
- **加载状态**：Skeleton Screen（骨架屏）替代传统的 Loading 图标

### Anti-Patterns to Avoid

**❌ 微信式布局（您当前的问题）**
- 底部标签栏在 PC 端显得拥挤
- 内容区域过窄，浪费屏幕空间
- 移动端优先的设计在 PC 端体验差

**❌ 过时的组件样式**
- 纯表格展示（应使用卡片式布局）
- 原始按钮样式（应使用圆角、渐变、阴影的现代按钮）
- 无反馈的交互（所有操作都应有视觉反馈）
- 单调的颜色（应使用渐变、阴影、微动画提升精致感）

**❌ 过度复杂的导航**
- 多级菜单让用户迷失
- 功能入口不明显
- 导航与内容竞争注意力

**❌ 信息过载的首页**
- 太多元素抢夺注意力
- 没有清晰的视觉层次
- 用户不知道应该先看哪里

**❌ 不一致的设计语言**
- 不同页面风格迥异
- 颜色、字体、间距不统一
- 用户需要重新学习每个页面

**❌ 静态无趣的页面**
- 无任何动效，页面死板
- 滚动时内容突然出现，无过渡
- 缺少视觉反馈，用户不确定操作是否生效

### Design Inspiration Strategy

**✅ 采用的设计模式（What to Adopt）：**

- **内容卡片布局**（Apple、n8n）→ 支持核心体验（浏览和阅读内容）
- **顶部极简导航**（GitHub、Apple）→ 符合 PC 优先的平台策略
- **状态颜色系统**（GitHub）→ 清晰传达角色和内容状态
- **代码高亮优化**（GitHub）→ 提升技术博客阅读体验
- **骨架屏加载**（Apple）→ 提升加载体验，减少用户焦虑

**🔧 适配的设计模式（What to Adapt）：**

- **滚动触发动画**（n8n、Apple）→ 简化为内容卡片渐入，避免过度复杂
- **Hover 反馈系统**（所有产品）→ 应用于卡片、按钮、链接，但保持克制
- **大尺寸英雄区域**（Apple）→ 简化为首页欢迎区域，突出个性化欢迎语
- **页面过渡动画**（Apple）→ 使用简单的淡入淡出，避免复杂的路由动画
- **渐变和阴影系统**（n8n、Apple）→ 建立统一的设计 token，确保一致性

**❌ 避免的设计模式（What to Avoid）：**

- **微信式底部导航** → 与 PC 优先策略冲突
- **纯表格展示** → 过时且不够现代
- **过度复杂的动画** → 影响性能和用户体验
- **不一致的组件样式** → 破坏专业感
- **静态无反馈的交互** → 降低用户信心

**设计原则总结：**

**1. 动效服务于体验，而非炫技**
- 所有动画都应有明确的目的（引导注意力、提供反馈、提升流畅感）
- 避免过度使用动画导致性能问题或视觉干扰
- 保持动画时长在 200-400ms 之间，符合人类感知

**2. 精致来自细节**
- 圆角、阴影、间距的统一规范
- 颜色系统的一致性（主色、辅助色、状态色）
- 字体层级清晰（标题、正文、辅助文字）

**3. 反馈是信任的基础**
- 每个用户操作都应有视觉反馈
- 加载状态明确，不让用户猜测
- 错误提示友好，引导用户解决问题

**4. 现代感来自留白和层次**
- 大量留白营造专业感
- 通过阴影和颜色建立视觉层次
- 内容优先，UI 元素退居次要

## Design System Foundation

### Design System Choice

**选定方案：Element Plus + TailwindCSS 深度定制**

采用现有的 Element Plus 组件库结合 TailwindCSS 进行深度样式定制，实现现代化、动态化的 PC 优先设计。

### Rationale for Selection

**选择理由：**

1. **零迁移成本**
   - 项目已在使用 Element Plus 和 TailwindCSS
   - 团队熟悉这两个库的使用方式
   - 无需引入新的依赖或学习新的 API

2. **强大的定制能力**
   - TailwindCSS 提供原子化样式，可以精确控制每个细节
   - Element Plus 提供丰富的功能组件（表单、验证、对话框、表格等）
   - 两者结合可以同时获得功能完整性和样式自由度

3. **符合设计灵感**
   - 通过 TailwindCSS 可以实现 Apple/n8n 风格的圆角、阴影、渐变
   - 可以添加自定义动画（滚动渐入、hover 效果、页面过渡）
   - 可以避免过时的组件样式（纯表格、原始按钮）

4. **适合团队能力**
   - 学生社团团队，需要控制学习成本
   - Element Plus 处理复杂逻辑，TailwindCSS 处理样式
   - 分工明确，开发效率高

5. **长期可维护性**
   - Element Plus 和 TailwindCSS 都是成熟的开源项目
   - 社区活跃，文档完善，问题容易解决
   - 升级路径清晰，技术债务可控

### Implementation Approach

**实施方法：**

**阶段 1：建立设计 Token 系统**
- 定义统一的颜色变量（主色、辅助色、状态色、角色色）
- 定义间距系统（4px 基数：4, 8, 12, 16, 24, 32, 48, 64）
- 定义圆角系统（4px, 8px, 12px, 16px, 9999px）
- 定义阴影系统（sm, md, lg, xl）
- 定义字体层级（标题、正文、辅助文字、代码）

**阶段 2：重写核心组件样式**
- **卡片组件**：圆角 + 阴影 + hover 效果（放大、阴影加深）
- **按钮组件**：圆角 + 渐变 + hover/active 状态
- **导航组件**：顶部极简导航 + 粘性定位 + 滚动阴影
- **标签组件**：角色标识（预备灰、正式蓝、负责人金）
- **内容状态标签**：草稿、待审核、已发布的颜色区分

**阶段 3：添加动效系统**
- **滚动渐入动画**：内容卡片进入视口时淡入 + 轻微上移
- **Hover 反馈**：卡片、按钮、链接的微妙反馈
- **页面过渡**：路由切换时的淡入淡出效果
- **加载状态**：骨架屏（Skeleton Screen）替代 Loading 图标
- **成功反馈**：转正通过的庆祝动画（confetti）

**阶段 4：逐个页面应用**
- 首页重构（欢迎区域 + 内容卡片列表）
- 内容详情页重构（标题、作者、富文本、代码高亮）
- 发布页面重构（极简编辑器 + 预览）
- 审核页面重构（审核队列 + 投票卡片）
- 个人中心重构（资料展示 + 内容列表）

### Customization Strategy

**定制策略：**

**1. Element Plus 组件覆盖策略**
- 保留功能复杂的组件：表单验证、对话框、表格、分页
- 重写样式敏感的组件：按钮、卡片、标签、导航
- 使用 TailwindCSS 类覆盖 Element Plus 默认样式
- 创建自定义组件包装器，统一应用设计 Token

**2. TailwindCSS 配置扩展**
- 在 `tailwind.config.js` 中定义自定义颜色、间距、圆角、阴影
- 创建自定义动画（fade-in、slide-up、scale-in）
- 定义组件类（.card、.btn-primary、.badge-role）
- 使用 `@layer` 指令组织自定义样式

**3. 动画实现方案**
- **滚动渐入**：使用 Intersection Observer API + CSS transitions
- **Hover 效果**：CSS transitions（transform、box-shadow）
- **页面过渡**：Vue Router 的 `<transition>` 组件
- **庆祝动画**：使用 canvas-confetti 库或自定义 CSS 动画
- **加载状态**：Element Plus 的 Skeleton 组件或自定义实现

**4. 设计 Token 管理**
- 使用 CSS 变量（Custom Properties）存储设计 Token
- 在 `:root` 中定义全局变量
- 在 TailwindCSS 配置中引用这些变量
- 确保颜色、间距、圆角在所有组件中一致

**5. 响应式策略**
- PC 优先：首先优化 1920x1080 和 1366x768
- 断点设置：sm (640px), md (768px), lg (1024px), xl (1280px), 2xl (1536px)
- 内容区域最大宽度：max-w-4xl (896px) 或 max-w-5xl (1024px)
- 不强制兼容移动端，但保证基本可用性
