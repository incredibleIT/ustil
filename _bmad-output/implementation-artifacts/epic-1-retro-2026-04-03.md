## 遇到的挑战

### 1. 逻辑删除字段遗漏
**问题：** Story 1.4 初始实现时，部分实体类未添加 @TableLogic 注解

**影响：** 需要创建额外的 Flyway 迁移脚本 V2__add_deleted_column.sql

**解决方案：** 代码审查发现后，统一为所有实体类添加 @TableLogic 注解

**教训：** 数据库设计阶段应使用检查清单，确保所有表包含标准字段（created_at, updated_at, deleted）

### 2. Spring Boot 3.5.0 与 MyBatis-Plus 版本兼容性问题
**问题：** 应用启动时报错 `Invalid value type for attribute 'factoryBeanObjectType': java.lang.String`

**根因分析：**
- Spring Boot 3.5.0 与 MyBatis-Plus 3.5.5 不兼容
- MyBatis-Plus 3.5.5 依赖的 mybatis-spring 2.1.2 不支持 Spring Boot 3.5.0

**解决方案：**
1. 升级 MyBatis-Plus 从 3.5.5 到 3.5.9
2. 排除 mybatis-spring 2.1.2，显式引入 mybatis-spring 3.0.4

**修改内容（pom.xml）：**
```xml
<!-- MyBatis-Plus -->
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-boot-starter</artifactId>
    <version>3.5.9</version>
    <exclusions>
        <exclusion>
            <groupId>org.mybatis</groupId>
            <artifactId>mybatis-spring</artifactId>
        </exclusion>
    </exclusions>
</dependency>
<dependency>
    <groupId>org.mybatis</groupId>
    <artifactId>mybatis-spring</artifactId>
    <version>3.0.4</version>
</dependency>
```

**教训：** 
- 技术栈版本需要验证兼容性
- Spring Boot 3.5+ 需要 MyBatis-Plus 3.5.9+ 和 mybatis-spring 3.0+

### 3. Flyway 迁移脚本版本冲突
**问题：** 存在两个 V2 版本迁移脚本（V2__seed_data.sql 和 V2__add_deleted_column.sql）

**解决方案：** 将 V2__add_deleted_column.sql 重命名为 V3__add_deleted_column.sql

**教训：** Flyway 迁移脚本版本号必须唯一，新增变更应使用递增版本号

### 测试过程

#### 1. 初始测试 - 发现权限问题
**请求:**
```bash
curl http://127.0.0.1:8080/api/test/success
```

**结果:** HTTP 401 Unauthorized

**问题分析:** Spring Security 默认配置阻止了匿名访问测试接口

#### 2. 修复 Security 配置

**创建 SecurityConfig.java:**
```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/test/**").permitAll()
                .requestMatchers("/api-docs/**", "/swagger-ui.html", "/swagger-ui/**").permitAll()
                .anyRequest().authenticated()
            )
            .httpBasic(AbstractHttpConfigurer::disable)
            .formLogin(AbstractHttpConfigurer::disable);
        return http.build();
    }
}
```

**更新 TestController:**
- 添加 `@PreAuthorize("permitAll()")` 注解到测试方法
