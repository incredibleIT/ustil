# 首页动态背景特效方案对比

##  目标
探索并实现多种高级动态背景特效，找到最适合"沈阳工业大学计算机程序设计社团"首页的视觉效果。

---

## 当前方案

### 方案 0: Three.js 旋转多面体（当前实现）
- **技术栈**: Three.js + 线框几何体
- **效果描述**: 多个旋转的线框多面体（二十面体、二十面体2、二十面体3），带有线框装饰
- **优点**: 
  - ✅ 科技感强，符合计算机社团定位
  - ✅ 性能较好，线框渲染开销小
  - ✅ 代码成熟稳定
- **缺点**:
  - ❌ 视觉元素单一
  - ❌ 缺乏层次感
- **适用场景**: 科技类、编程类网站

## 🆕 待探索方案

### 方案 1: 粒子连线网络（Particle Network）⭐ 推荐

#### 效果描述
- 大量粒子在屏幕上自由漂浮
- 当粒子距离足够近时，自动连线形成网络
- 粒子会跟随鼠标移动产生互动
- 整体呈现出"神经网络"或"星座"的视觉效果

#### 技术实现
- **方案 A**: Canvas 2D 原生绘制（推荐，性能最佳）
- **方案 B**: Three.js Points + Lines（3D 效果）
- **关键参数**: 
  - 粒子数量: 80-120 个
  - 连线距离阈值: 120-150px
  - 粒子移动速度: 0.3-0.6 px/frame
  - 连线透明度: 距离越近越不透明

#### 视觉效果
```
•──────•──────•
  ╲    │    ╱
    •──•──•
  ╱    │    ╲
•──────•──────•
```

#### 优点
- ✅ 极具科技感和未来感
- ✅ 互动性强（鼠标跟随）
- ✅ 视觉层次丰富
- ✅ 性能优秀（Canvas 2D）
- ✅ 适合编程/科技主题

#### 缺点
- ⚠️ 移动端粒子数量需要减少
- ⚠️ 需要处理窗口resize

#### 参考网站
- https://particles.js.org/
- https://vincentgarreau.com/particles.js/
- GitHub 首页背景

### 方案 2: 流动渐变波浪（Gradient Waves）

#### 效果描述
- 多层半透明波浪形状缓慢流动
- 使用 CSS Gradient 或 SVG 实现
- 波浪之间有视差滚动效果
- 颜色缓慢渐变循环

#### 技术实现
- **方案 A**: SVG Path + CSS Animation（推荐）
- **方案 B**: Canvas 绘制贝塞尔曲线
- **关键参数**:
  - 波浪层数: 3-4 层
  - 每层速度: 8s-20s 循环
  - 振幅: 50-100px
  - 颜色: 蓝紫渐变循环

#### 视觉效果
```
╔════════════════════════════╗
║  ~~~~~~波浪1（缓慢）~~~~~~  ║
║    ~~~~波浪2（中等）~~~~    ║
║      ~~波浪3（快速）~~      ║
╚════════════════════════════╝
```

#### 优点
- ✅ 视觉柔和，舒适度高
- ✅ 性能极佳（CSS/SVG）
- ✅ 移动端友好
- ✅ 不会干扰内容阅读
- ✅ 现代感强

#### 缺点
- ❌ 科技感不如粒子效果
- ❌ 缺乏互动性

#### 参考网站
- https://www.apple.com/ (部分页面)
- Stripe 官网背景
- https://cssgradient.io/

### 方案 3: 几何图形阵列（Geometric Grid）

#### 效果描述
- 网格状排列的几何图形（正方形、六边形、三角形）
- 图形会周期性地进行变换（缩放、旋转、颜色变化）
- 鼠标悬停时图形会产生涟漪效应
- 整体呈现出"像素化"或"矩阵"效果

#### 技术实现
- **方案 A**: CSS Grid + CSS Animation（推荐，简单）
- **方案 B**: Canvas 绘制（高性能）
- **关键参数**:
  - 网格大小: 40-60px
  - 图形类型: 正方形/六边形/三角形
  - 动画周期: 2-5s
  - 涟漪半径: 3-5 个格子

#### 视觉效果
```
□ □ ■ □ □
□ ■ ■ ■ □
■ ■ ■ ■ ■
□ ■ ■ ■ □
□ □ ■ □ □
（鼠标中心的涟漪效果）
```

#### 优点
- ✅ 科技感强，类似"矩阵"
- ✅ 互动性极佳（涟漪效应）
- ✅ 可自定义程度高
- ✅ 视觉冲击力强

#### 缺点
- ⚠️ 需要较多的 DOM 元素（CSS 方案）
- ⚠️ 移动端需要简化

#### 参考网站
- https://grid.gallery/
- https://bruno-simon.com/ (部分效果)

### 方案 4: 流体变形（Fluid Morphing）

#### 效果描述
- 多个不规则的流体形状在屏幕上漂浮
- 形状会不断变形（morphing）
- 颜色缓慢渐变
- 流体之间会相互影响（排斥/吸引）

#### 技术实现
- **方案 A**: SVG Path Morphing（推荐）
- **方案 B**: Canvas + 噪声算法（Perlin Noise）
- **关键参数**:
  - 流体数量: 3-5 个
  - 变形速度: 3-8s 循环
  - 移动速度: 0.2-0.4 px/frame
  - 颜色: 蓝紫色系渐变

#### 视觉效果
```
  ╱╲
 ╱  ╲
( 流体1 )  ← 不断变形
 ╲  ╱
  ╲╱
```

#### 优点
- ✅ 视觉效果独特，艺术感强
- ✅ 柔和舒适
- ✅ 现代感十足
- ✅ 不会显得拥挤

#### 缺点
- ❌ 性能开销较大（SVG morphing）
- ❌ 移动端适配复杂
- ❌ 科技感不如粒子/几何

#### 参考网站
- https://www.awwwards.com/ (部分获奖作品)
- https://fluidsimulation.io/

### 方案 5: 代码雨/字符矩阵（Code Rain）

#### 效果描述
- 类似《黑客帝国》的代码雨效果
- 随机字符从顶部下落
- 字符会淡出消失
- 可以自定义字符集（代码片段、编程关键词等）

#### 技术实现
- **方案 A**: Canvas 绘制（推荐，性能最佳）
- **方案 B**: DOM 元素 + CSS Animation
- **关键参数**:
  - 列数: 40-60 列
  - 字符集: 编程关键词、符号、数字
  - 下落速度: 2-5 px/frame
  - 字符大小: 14-16px
  - 颜色: 蓝紫色渐变（非传统绿色）

#### 视觉效果
```
| 0 1 { } |
| i f ( ) |
| 0 1 { } |
| c o d e |
| 0 1 { } |
（垂直下落的字符流）
```

#### 优点
- ✅ 极具编程/科技感
- ✅ 与"计算机程序设计社团"主题高度契合
- ✅ 视觉冲击力强
- ✅ 互动性强（可加入鼠标跟随）

#### 缺点
- ⚠️ 可能干扰内容阅读（需要调整透明度）
- ⚠️ 移动端需要减少列数
- ⚠️ 不适合所有类型的网站

#### 参考网站
- https://matrix-code-rain.vercel.app/
- 黑客帝国主题网站

### 方案 6: 动态声波/音频可视化（Audio Visualizer Style）

#### 效果描述
- 类似音频频谱的波形动画
- 多个波形条周期性起伏
- 可以有多个波形层叠加
- 波形会根据"虚拟音频"产生变化

#### 技术实现
- **方案 A**: Canvas 绘制正弦波（推荐）
- **方案 B**: SVG Path + CSS Animation
- **关键参数**:
  - 波形层数: 3-5 层
  - 波峰数量: 8-12 个
  - 振幅: 30-80px
  - 频率: 0.5-2 Hz
  - 颜色: 蓝紫色渐变

#### 视觉效果
```
╱╲  ╱╲  ╱╲
  ╲╱  ╲╱  ╲╱
（周期性起伏的波形）
```

#### 优点
- ✅ 视觉动感强
- ✅ 科技感十足
- ✅ 性能优秀（Canvas）
- ✅ 可以加入真实音频互动

#### 缺点
- ❌ 可能显得过于"动态"
- ❌ 缺乏与编程主题的直接关联

#### 参考网站
- https://wavesurfer-js.org/
- 音乐类网站背景

## 💡 实施建议

### 方案选择策略
1. **先实现方案 1（粒子连线）** → 快速验证效果
2. **再实现方案 5（代码雨）** → 对比两种编程主题特效
3. **根据反馈选择** → 保留效果更好的方案
4. **可选优化** → 尝试方案 3（几何阵列）作为备选

### 技术要点
- 所有方案都使用 Canvas 2D 实现（性能最佳）
- 保持浅色调主题（蓝紫色系）
- 添加鼠标互动效果
- 移动端自适应（减少粒子/字符数量）
- 保持与 GSAP/Lenis 的兼容性

### 测试标准
- ✅ 视觉效果评分（1-10）
- ✅ 性能测试（FPS > 55）
- ✅ 移动端适配测试
- ✅ 与内容可读性测试
- ✅ 用户偏好调查
